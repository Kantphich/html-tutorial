# บทที่ 6: การแบ่งสัดส่วนและ Semantic HTML (Layout & Semantic Tags)

เมื่อหน้าเว็บเราเริ่มมีเนื้อหาจำนวนมาก การโยนทุกอย่างลงใน `<body>` เฉยๆ จะทำให้โค้ดอ่านยากขึ้น และตัวค้นหา (เช่น Google) ก็ไม่รู้ว่าส่วนไหนคือเนื้อหาหลัก ส่วนไหนคือเมนู ในบทนี้เราจะมาเรียนรู้การจัดกลุ่มเนื้อหาให้เป็นหมวดหมู่กันครับ

---

## 1. การจัดกลุ่มแบบมาตรฐาน `<div>` และ `<span>`
แท็กสองตัวนี้ถือเป็นแท็ก "สารพัดประโยชน์" ที่เรามักใช้จัดกลุ่มเนื้อหาเพื่อรอการตกแต่งด้วย CSS (ที่จะเรียนในบทต่อไป)

- `<div>` (Division): เป็นแท็กแบบ **Block** หมายความว่าเนื้อหาข้างในจะจองพื้นที่หน้าจอ 1 บรรทัดเต็มๆ เสมอ (ใช้คลุมเป็นกล่องใหญ่ๆ)
- `<span>`: เป็นแท็กแบบ **Inline** จะจองพื้นที่หน้าจอเฉพาะเท่าที่ข้อความยาวเท่านั้น (มักใช้คลุมข้อความสั้นๆ เพื่อเปลี่ยนสีข้อความเฉพาะคำ)

**ตัวอย่าง:**
```html
<body>
    
    <div id="product-card">
        <h3>รองเท้าผ้าใบ</h3>
        <p>ราคาสินค้า: <span style="color: red;">990 บาท</span></p>
    </div>

    <div id="product-card-2">
        <h3>เสื้อยืด</h3>
        <p>ราคาสินค้า: <span style="color: red;">350 บาท</span></p>
    </div>

</body>
```

---

## 2. Semantic HTML คืออะไร?
`<div>` นั้นมีประโยชน์มาก แต่มันไม่มี "ความหมาย" อะไรเลย (ไม่มี Semantic) เพื่อแก้ปัญหานี้ HTML5 จึงเพิ่มแท็กที่มีความหมายในตัวมันเองเข้ามา เพื่อให้หน้าเว็บมีโครงสร้างที่ชัดเจนขึ้น เช่น:

- `<header>`: ส่วนหัวของหน้าเว็บหรือเนื้อหา (มักใส่โลโก้หรือชื่อเว็บ)
- `<nav>` (Navigation): ส่วนที่เป็นเมนูลิงก์นำทาง
- `<main>`: ส่วนเนื้อหาหลักที่สำคัญที่สุดในหน้านั้น (มีได้แค่อันเดียวต่อหน้า)
- `<section>`: ส่วนย่อยๆ ของเนื้อหา ใช้แบ่งหน้าเว็บเป็นตอนๆ
- `<article>`: บทความ หรือเนื้อหาที่อ่านจบได้ในตัวเอง
- `<aside>`: แถบด้านข้าง (Sidebar) หรือเนื้อหาที่ไม่ใช่เรื่องหลัก
- `<footer>`: ส่วนท้ายของหน้าเว็บ หรือท้ายของแต่ละส่วน

---

## Step-by-step: การปรับโครงสร้างเว็บไซต์ให้เป็น Semantic

เรามาจำลองการสร้างหน้า "บทความข่าว" ง่ายๆ ให้มีโครงสร้างที่ถูกต้องกันครับ ให้เปิดไฟล์ `index.html` แล้วพิมพ์โค้ดด้านล่างนี้ลงไป

### Step 1: โครงสร้างหลัก (Header, Main, Footer)
สร้างโครงสร้างชั้นนอกสุดก่อน:

```html
<body>
    
    <!-- ส่วนหัวของเว็บไซต์ -->
    <header>
        <h1>เว็บไซต์ข่าวของฉัน</h1>
    </header>

    <!-- เมนูนำทาง -->
    <nav>
        <ul>
            <li><a href="#">หน้าแรก</a></li>
            <li><a href="#">ข่าวไอที</a></li>
            <li><a href="#">ติดต่อเรา</a></li>
        </ul>
    </nav>

    <!-- เนื้อหาหลักของหน้า -->
    <main>
        
        <!-- เราจะใส่ข่าวลงตรงนี้ -->

    </main>

    <!-- ส่วนท้ายของเว็บไซต์ -->
    <footer>
        <p>&copy; 2026 เว็บไซต์ข่าวของฉัน สงวนลิขสิทธิ์</p>
    </footer>

</body>
```

### Step 2: ใส่เนื้อหาบทความด้วย Section และ Article
ในส่วนของ `<main>` เราจะเพิ่มส่วนของเนื้อหาข่าวลงไป:

```html
    <main>
        
        <section id="latest-news">
            <h2>ข่าวล่าสุด</h2>
            
            <article>
                <h3>การเปิดตัว HTML เวอร์ชั่นใหม่!</h3>
                <p>นักพัฒนาทั่วโลกตื่นเต้นกับฟีเจอร์ใหม่...</p>
                <a href="#">อ่านต่อ...</a>
            </article>

            <article>
                <h3>Cypress ทำให้การเทสเป็นเรื่องง่ายขึ้น</h3>
                <p>บทความสัมภาษณ์นักทดสอบระบบชั้นนำ...</p>
                <a href="#">อ่านต่อ...</a>
            </article>
        </section>

    </main>
```
*ลองเซฟแล้วดูหน้าเว็บครับ หน้าตาอาจจะดูธรรมดา เพราะเรายังไม่ได้จัดหน้าด้วย CSS แต่ในระดับโครงสร้าง (Structure) ถือว่าเว็บนี้ยอดเยี่ยมมาก!*

## Cypress Focus: Semantic ช่วยอะไรในการเทส?

เวลาที่เราเขียน Automated Test เราต้องระบุขอบเขตให้ชัดเจน สมมติว่าหน้าเว็บเรามีปุ่มที่มีคำว่า "อ่านต่อ..." อยู่เต็มไปหมด

ถ้าหน้าเว็บเรามีโครงสร้าง Semantic ที่ดี หรือมีการใช้ `<div id="...">` แบ่งโซนไว้ Cypress จะทำงานง่ายมาก:

```javascript
// ตัวอย่างคำสั่งให้ Cypress เข้าไปหาปุ่มในข่าวล่าสุดเท่านั้น
cy.get('section#latest-news').within(() => {
    // หาปุ่ม 'อ่านต่อ' เฉพาะที่อยู่ในโซนข่าวล่าสุด 
    cy.contains('อ่านต่อ...').click()
})
```
จะเห็นได้ว่า การแบ่งสัดส่วนหน้าเว็บด้วย Tag เหล่านี้ ไม่ได้แค่ทำให้โค้ดเราเป็นระเบียบ แต่ยังช่วยให้ Test Script ทำงานได้ถูกต้องไม่ผิดพลาดด้วย

> โครงสร้าง HTML ครบถ้วนแล้ว! ต่อไปได้เวลาทำให้มันสวยงาม ด้วย **บทที่ 7: ตกแต่งหน้าเว็บด้วย CSS เบื้องต้น** ครับ
